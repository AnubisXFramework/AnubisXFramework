We have received several questions about the AnubisX Framework. Here are the answers.

**Q: Is this a software product I can download?**

A: No. The AnubisX Framework is a complete theoretical specification — documents, definitions, equations, and designs. Implementation code is not included. We have published the full specification for researchers and developers to build upon.

**Q: Does it work? Has it been tested?**

A: The framework specifies experiments and benchmarks designed to test every component. These have not yet been fully executed. The framework is a pre-implementation design specification. All acceptance criteria are provisional pending empirical calibration.

**Q: What makes this different from existing behavioral analysis methods?**

A: Three key differences: (1) formal axiomatic foundation defining the logical structure for behavioral identity attribution; (2) multi-modal integration — five behavioral modalities under one unified framework with defined fusion strategies; (3) quantified uncertainty — every conclusion includes a measured confidence estimate with defined acceptance criteria.

**Q: Is this about mass surveillance?**

A: No. The framework is designed for forensic investigation contexts with appropriate legal oversight. It embeds the P³ Protocol (Proportionality and Privacy Protection) as a core constraint — attribution methods must be proportional to investigative need, and data collection follows the minimum necessary principle.

**Q: Who is this for?**

A: Forensic investigators seeking rigorous methodology, cybersecurity analysts tracking threat actors, academic researchers in behavioral forensics, legal professionals evaluating digital evidence, and policy makers developing digital evidence standards.

**Q: Can I use this commercially?**

A: The specification is available under CC BY 4.0. Commercial implementations are permitted with attribution.

**Q: What's next?**

A: Software implementation and empirical validation, followed by peer-reviewed publication.
