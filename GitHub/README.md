## Overview

The AnubisX Framework is a formal scientific methodology for determining the identity of a human operator from their digital behavioral patterns. Unlike traditional attribution methods that rely on transient technical artifacts — IP addresses, device fingerprints, browser configurations — the framework measures **persistent human cognitive signatures** that are rooted in stable cognitive processing habits.

The framework is a **complete theoretical specification** with a **validated prototype implementation**. The theoretical layer defines the axiomatic foundation, mathematical formalization, algorithm specifications, validation methodology, benchmark protocols, experiment designs, and case studies. The prototype layer implements a subset of the framework's stylometric fingerprinting and similarity search capabilities.

## Core Concepts

| Concept | Definition |
|---|---|
| **Cognitive Centroid** | The theoretical attractor of an individual's behavioral patterns in feature space |
| **Five Behavioral Modalities** | Forensic Stylometry, Chrono-Profiling, Terminal Execution Profiling, Relational Network Analysis, Environmental Media Forensics |
| **Multi-Modal Fusion** | Combining evidence from independent behavioral signals |
| **Likelihood Ratio (LR)** | Statistical quantification of evidence strength |
| **Identity Intelligence (IdINT)** | A proposed formal forensic discipline for behavioral attribution |

## How the Pipeline Works

The framework processes behavioral data through six defined stages:

```
Ingestion → Feature Extraction → Profile Construction → Comparison → Evidence Evaluation → Decision
```

1. **Ingestion**: Raw digital traces are collected, validated, and metadata extracted
2. **Feature Extraction**: Modality-specific processors transform raw data into feature vectors
3. **Profile Construction**: Features are aggregated into behavioral profiles
4. **Comparison**: Profiles are compared using similarity functions
5. **Evidence Evaluation**: Scores are calibrated to Likelihood Ratios and fused across modalities
6. **Decision**: Attribution conclusions are produced with quantified uncertainty

Three workflows are supported: **Identification**, **Verification**, and **Forensic Comparison**.
